Core Approach

My Data Philosophy

Clean Data First

Accurate analysis starts with clean data. I focus on careful data validation and organized queries to ensure that every dashboard and report I build is built on a reliable, trustworthy foundation.

Stakeholder Clarity

Data is most valuable when everyone can understand it. I bridge the gap between complex datasets and business teams, focusing on building clear, intuitive dashboards that help end-users find answers quickly.

Practical Automation

By utilizing Python and SQL to script repetitive reporting tasks, I try to eliminate manual data entry wherever possible. Doing this helps teams get their time back so they can focus on actual analysis.

Bridging the gap between engineering and executive strategy.

With a foundation in Computer Science and an active M.Sc in Data Science, my approach is deeply analytical. I don't just build dashboards; I engineer the underlying SQL queries, automate the Python pipelines, and ensure data integrity upstream before it reaches leadership.

My ethos is simple: data should reduce friction, not create it. Whether it's cutting 20 hours of manual Excel formatting down to an automated cron job, or designing Metabase views that align with rigid KPI frameworks, I focus on delivering scalable, high-impact business value.
