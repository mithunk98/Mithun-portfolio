The Problem

A retail client's data was fragmented across three separate systems: a product catalogue database, a customer reviews platform (unstructured text), and a sales transaction system. Each team accessed only their own silo, making cross-signal decisions — like adjusting pricing based on review sentiment combined with sales velocity — essentially impossible without manual extraction and ad-hoc analysis.

The business goal was to automate pricing and promotional decisions based on predicted product ratings, removing the manual review cycle that was slowing the merchandising team by several days each week.

"We had all the data we needed. It was just in three different places and nobody had connected it."

Data Architecture

The first phase was building a centralised data lake on AWS S3 using Boto3 to ingest from all three source systems:

  • Product catalogue exports (structured CSV) — ingested nightly via scheduled Boto3 upload script
  • Customer reviews (JSON from platform API) — streamed into S3 with metadata tagging by product ID
  • Sales transactions (PostgreSQL dump) — exported and partitioned by week into S3 Parquet files

A unified product ID key linked records across all three sources, enabling the first ever cross-silo join in the business's history.

Model Pipeline

1
NLP Feature Engineering

Applied TF-IDF vectorisation on customer review text to extract sentiment signals. Engineered features including review length, keyword frequency (quality, defect, recommend), and average star rating per product.

2
Numerical Feature Engineering

Created structured features from sales and catalogue data: sales velocity (units/week), price elasticity index, return rate, stock-out frequency, and category margin contribution.

3
Random Forest Classifier

Combined NLP and numerical features into a single feature matrix. Trained a Random Forest classifier to predict product rating tier (High / Medium / At-Risk). Tuned via 5-fold cross-validation; selected max_depth and n_estimators based on F1 score.

4
Decision Trigger Layer

Model predictions fed into a rule engine: At-Risk products above a sales volume threshold trigger an automated promotional flag; High-rated products with low visibility trigger a merchandising boost recommendation.

Key Challenges

  • Class imbalance — only 12% of products fell into the At-Risk tier. Used SMOTE oversampling during training to prevent the model from ignoring the minority class.
  • Review sparsity — 30% of products had fewer than 5 reviews, making NLP features unreliable. Imputed sparse NLP features with category-level averages rather than dropping products entirely.
  • Feature leakage — initial model was inadvertently using the current star rating as an input. Caught during validation; rebuilt feature set to use only pre-rating signals.

Results

  • 25% improvement in predictive accuracy vs. the baseline (sales-only model)
  • Three data silos consolidated into a single S3-based data lake
  • Decision trigger layer automated merchandising actions that previously required manual analyst review
  • Merchandising team's weekly review cycle cut from 3 days to same-day

What I'd Do Differently

I'd implement a proper MLflow experiment tracking setup from the start — managing model versions in notebooks became unwieldy after the third iteration. I'd also add a data quality layer on the S3 ingestion step to catch upstream schema changes before they silently corrupt the feature pipeline.
